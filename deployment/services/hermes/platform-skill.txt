# MTE Platform Operations

Use Hermes' native terminal tool. The dedicated service receives a narrow
projection of the credentials it needs; never read or print
`/root/.config/mte-secrets/platform.env` or dump the process environment.

```bash
python3 /opt/mte-platform/bin/server-verify.py status
python3 /opt/mte-platform/bin/server-hermes.py status
systemctl status mte-hermes.service --no-pager
```

Use application APIs directly when an API operation is needed. Pass tokens by
environment variable and keep response output limited to non-secret status or
resource IDs. Prefer existing platform commands over handwritten shell logic:

```bash
python3 /opt/mte-platform/bin/server-hermes.py health
python3 /opt/mte-platform/bin/server-verify.py status
python3 /opt/mte-platform/bin/server-config.py diff
```

Before a destructive or externally visible action, explain it to the authorized
user and use Hermes' built-in command approval flow. Make repairs reproducible
in the platform source, then run the relevant native status/health command and
report each component state exactly.
